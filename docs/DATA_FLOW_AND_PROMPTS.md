# From HPC results to the report: data flow & LLM prompts

How MetaMAVS turns two GOTTCHA2 `.tsv` files fetched from Sapelo2 into
`report.md` / `report.html`, and exactly where (and how) the LLM is involved.
All references are to the code as verified for the `Genome_72` run.

> **Two things people usually get wrong**
> 1. The report is **not** written by the LLM. `report_writer_agent` is plain,
>    deterministic Python; the LLM only contributes one *section* and a few
>    reasoning strings.
> 2. The LLM never sees the raw `.tsv`. It sees **already-computed structured
>    summaries** rendered into a prompt.

---

## 1. What comes back from the HPC

Only two files are transferred back per sample (see
[`REAL_DATA_VALIDATION.md`](REAL_DATA_VALIDATION.md) / the transfer code in
`metamavs/remote/backends.py` and `metamavs/agents/result_sync_agent.py`):

| Local file | Role | Required? |
|---|---|---|
| `results/raw/gottcha2/<sid>.gottcha2.tsv` | the hit table (NAME / READ_COUNT / TAXID / LEVEL) | **yes** — the actual data |
| `results/raw/gottcha2/<sid>.gottcha2.lineage.tsv` | lineage, used to keep only *Viruses*-superkingdom taxa | optional — parser degrades gracefully without it (`parsers/gottcha2.py:54-55`) |

Declared as the job's `output_files` in `metamavs/remote/jobgen.py:158`; fetched
with `rsync -avz --partial --checksum` over the reused SSH ControlMaster
connection (`backends.py:71-88`).

## 2. The deterministic chain (tsv → structured summaries)

Between the raw `.tsv` and the LLM sit several **deterministic** agents. The LLM
does not run here — these compute the numbers.

```
Genome_72.gottcha2.tsv + .lineage.tsv
   │
   ▼  parse_gottcha2()                     metamavs/parsers/gottcha2.py
      → raw_viral_hits  (filtered to viral species; bacteria/archaea dropped)
   │
   ▼  tool_output_parser_agent             overwrites synthetic placeholders with REAL rows
   │
   ▼  taxonomy_agent                        classify phage / pathogen / false-positive → taxonomy_summary
   │
   ▼  abundance_agent                       RPM normalisation + trend → trend_summary
   │
   ▼  risk_assessment_agent                 assign Low/Medium/High/Critical → risk_summary  (High×2 here)
   │
   ▼  llm_interpretation_agent   ◄── the LLM sits HERE, reading the summaries above
   │
   ▼  report_writer_agent                   deterministic: assembles report.md / report.html
```

> Some of the intermediate agents (`taxonomy`, `abundance`, `risk`) *also* call
> the LLM to add a reasoning string, but only ever to interpret/annotate their
> own already-computed table — never to recompute the numbers. QC, host removal
> and novel-virus screening were disabled for the Sapelo2 config.

---

## 3. Where the LLM actually sits

The LLM (Gemma or Claude, selected by `LLM_BACKEND`) contributes **only**:

- the **AI-Assisted Interpretation** narrative section, and
- short `[LLM]` reasoning strings inside the QC / trend / risk tables.

It writes **none** of the report scaffold. `report_writer_agent` has **no LLM
call**; it concatenates sections from the state tables and inserts the
already-generated narrative text:

- `report_writer_agent.py:156-163` — inserts `llm_narrative["text"]` under
  `## AI-Assisted Interpretation`, attributed `_Generated by <model>_`.
- `report_writer_agent.py:74-75, 118-119` — inserts the QC / trend `[LLM]` notes.
- `report_writer_agent.py:214-219` — writes `report.md`, then renders `report.html`.

So "Claude/Gemma produced the report" really means: **the model wrote the
interpretation paragraph that the deterministic writer embedded.**

## 4. The prompts (llm_interpretation example)

Every LLM node has its own system + user prompt. `generate()`
(`metamavs/llm/client.py`) sends three pieces for the narrative:

### (a) System prompt — role + hard caution rules
`SYSTEM_PROMPT`, `metamavs/llm/prompts.py:16`. It fixes the role and the
non-negotiable scientific-caution rules, e.g.:

- "These are **DETECTED SEQUENCE SIGNALS**, never confirmed infections or outbreaks."
- "Treat anything below ~10 supporting reads as weak/uncertain."
- "Report bacteriophages separately from human/animal pathogens."
- "For every High/Critical detection, recommend orthogonal confirmatory testing."
- "Do not invent taxa, numbers, or trends not present in the provided data."
- Output exactly four sections, under ~450 words.

> This is *why* both backends' narratives are cautious and confirmation-first —
> it's enforced by the prompt, not model discretion.

Key line, verbatim:
> "You are given structured, already-computed results — your job is to interpret
> them for a public-health audience, **not to recompute them**."

### (b) User prompt — the run's structured summaries as JSON
`build_user_prompt(state)`, `metamavs/llm/prompts.py:183`. It renders the
computed summaries into a compact JSON payload — **not** the raw tsv:

- `run_name`, `samples` (n / locations / collection_dates / type)
- `detection_tools`, `overall_risk`, `risk_level_counts`
- `top_risk_taxa` (name, risk_level, total_reads, trend, reasons)
- `taxonomy` (n_taxa, n_phage, n_flagged_false_positive)
- `novel_candidates`, `recommended_actions_deterministic`

Prefixed with "Interpret the following MetaMAVS surveillance results. **Use only
this data.**" (This is why Claude's summary could cite "sapelo2 · 2025-12-02 ·
GOTTCHA2 · two High" — those fields are in the JSON.)

### (c) Cached prefix — shared reference
`cached_prefix = SHARED_REFERENCE` (`metamavs/llm/reference.py`): a curated
pathogen-background / conventions block reused across agents as the cache prefix,
plus per-taxon `pathogen_snippets` appended as background notes.

---

## 5. One-line summary

**Two tsv → a chain of deterministic agents computes the numbers → the LLM writes
only an interpretation paragraph from those summaries under strict caution
prompts → the deterministic `report_writer` embeds that paragraph into the
assembled report.** The LLM neither reads the tsv directly nor generates the
report as a whole.
